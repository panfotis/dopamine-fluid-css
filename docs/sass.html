Sass Addon dp.fluid()

An optional Sass function that gives you the same fluid clamp() math as Dopamine's utility classes — for use in custom SCSS where you can't add classes to the HTML.

This addon is independent from the core and from components. You don't need it to use Dopamine classes.

When to use it

Use Dopamine classes

When you control the HTML:

<h1 class="fs-32-72 mb-16-40">Title</h1>

Use dp.fluid()

When you can't add classes (Drupal renders it):

.field--body h1 {
  font-size: dp.fluid(32, 72);
  margin-bottom: dp.fluid(16, 40);
}

Setup

Option 1: Import the addon directly

@use 'dopamine-fluid/addons/sass/functions' as dp;

.my-component {
  font-size: dp.fluid(16, 48);
  padding: dp.fluid(12, 32);
}

Option 2: Use the auto-generated file

When you output to .scss, Dopamine generates a _dopamine-functions.scss alongside the utility classes with your config's viewport defaults:

dopamine ./templates --ext twig --out ./scss/_dopamine.scss
# Creates: _dopamine.scss + _dopamine-functions.scss
@use 'dopamine-functions' as dp;

.node--article .field--body p {
  font-size: dp.fluid(16, 48);
  margin-bottom: dp.fluid(8, 24);
}

Function signature

@function fluid($min, $max, $vpMin: 320, $vpMax: 1440)

// $min    — minimum value in pixels
// $max    — maximum value in pixels
// $vpMin  — viewport min (default from config)
// $vpMax  — viewport max (default from config)
// Returns — clamp(minRem, calc(Xvw + Yrem), maxRem)

Same math as the utility classes. Values in pixels, output in rem.

Examples

Basic usage

@use 'dopamine-functions' as dp;

h1 { font-size: dp.fluid(32, 72); }
p  { font-size: dp.fluid(16, 20); }
.hero { padding: dp.fluid(24, 80); }

Custom viewport override

// Scale between 480px and 1920px instead of default
.hero-title {
  font-size: dp.fluid(32, 96, 480, 1920);
}

Drupal theme example

@use 'dopamine-functions' as dp;

// CKEditor content — can't add classes
.text-formatted {
  h2 { font-size: dp.fluid(24, 48); margin-bottom: dp.fluid(16, 32); }
  h3 { font-size: dp.fluid(20, 32); margin-bottom: dp.fluid(12, 24); }
  p  { font-size: dp.fluid(16, 20); margin-bottom: dp.fluid(8, 16); }
  ul, ol { padding-left: dp.fluid(16, 32); }
}

Independence

The Sass addon is completely independent:

Core only

Scan HTML, generate CSS. No Sass needed.

Core + Sass

Classes in HTML + dp.fluid() for custom styles.

Core + Components

Classes + pre-built structure. No dp.fluid() needed.
